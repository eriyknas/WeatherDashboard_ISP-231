# Лабораторная работа №17-18. Корутины на практике: Метеосводка
## Описание
WeatherDashboard — это учебное Android-приложение, демонстрирующее работу с корутинами Kotlin в Jetpack Compose. Приложение имитирует загрузку данных о погоде из трёх независимых источников и показывает преимущества параллельных асинхронных операций перед последовательными.
## Функциональность приложения
- Загрузка температуры, влажности и скорости ветра с имитацией сетевых задержек
- Параллельная загрузка данных с помощью async
- Отображение индикаторов загрузки для каждого параметра
- Кнопка ручного обновления с защитой от повторного нажатия
- Имитация ошибок сети с корректной обработкой без краша
- Автоматическое обновление данных каждые 10 секунд с автоматической остановкой при закрытии экрана
- Отображение текстового прогресса загрузки
## Технологии и библиотеки
- Язык Kotlin
- UI Jetpack Compose
- Управление состоянием ViewModel + StateFlow
## Контрольные вопросы
1)
1. Когда использовать:
- launch — для операций, результат которых не нужен (логирование, запись в БД, отправка аналитики)
- async — когда нужен результат (загрузка данных, вычисления, парсинг)
2. Пример кода:
```kotlin
viewModelScope.launch {
    repository.logUserAction("refresh_clicked")
}
val weatherData = viewModelScope.async {
    repository.fetchFullWeather()
}.await()
```
2)
Suspend функция — функция, которая может приостановить своё выполнение, не блокируя поток, и возобновиться позже.
1. Нет, suspend функции можно вызывать только из другой suspend функции или из корутины (например, внутри launch или async).
2. delay() — это suspend функция, которая приостанавливает только текущую корутину, а поток освобождается для выполнения других задач. Thread.sleep() же блокирует весь поток полностью.
3)
1. Таблица диспетчеров:

| Диспетчер | Назначение | Пример задачи|
|:----:|:----: |:----:|
|Dispatchers.Main | UI-поток, обновление интерфейса | Изменение текста в Text(), показ Snackbar|
|Dispatchers.IO | Ввод-вывод, сеть, БД | Загрузка картинки из интернета, чтение файла|
|Dispatchers.Default | CPU-интенсивные вычисления | Сортировка большого массива, обработка изображений|
2. UI "зависнет" — приложение перестанет реагировать на нажатия, появится ANR (Application Not Responding), так как главный поток занят вычислениями и не может обрабатывать события интерфейса.
4)
1. Если исключение не обработать внутри корутины:
- При использовании launch — приложение упадёт с FATAL EXCEPTION
- При использовании async — исключение "всплывёт" только при вызове await(), но если оно произошло до этого, родительская корутина может отмениться
2. Без try-catch любое непойманное исключение в корутине упадёт в CoroutineExceptionHandler (если он не задан) и вызовет краш. try-catch позволяет обработать ошибку локально и показать её пользователю, не закрывая приложение.
5)
1. viewModelScope — это встроенная область видимости для корутин, привязанная к жизненному циклу ViewModel. Все корутины, запущенные в этом скоупе, автоматически отменяются при вызове onCleared() у ViewModel.
2.
- При закрытии экрана (Activity/Fragment уничтожается)
- При повороте экрана (ViewModel пересоздаётся? Нет — ViewModel выживает, но старая Activity уничтожается. Однако корутины в viewModelScope продолжают работать, так как ViewModel жива)
- Важно: при вызове viewModelScope.cancel() (редко) или при завершении родительской корутины
6) Как запустить проект
1. Клонируйте репозиторий
2. Откройте проект в Android Studio
3. Дождитесь синхронизации Gradle и нажмите Run
